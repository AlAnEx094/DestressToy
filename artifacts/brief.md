# Project Brief — DeStressToys

## Product type
- landing page (одностраничник + отдельная страница Privacy Policy)

## Goal
- collect leads: посетитель оставляет заявку через форму и получает концепт бренд-объекта под свою задачу. Оплаты на этом этапе нет.

## Audience
- B2B: маркетологи, HR-команды, event-менеджеры, закупщики корпоративных подарков
- Контекст принятия решения: готовят промонаборы, welcome kit, подарки партнерам, выставочные материалы, рассылки клиентам
- Ключевая боль: обычный мерч теряется, не запоминается, не остается на столе
- Уровень: профессиональные покупатели, не розница

## Core action
- Главный CTA: **Запросить концепт**
- Финальная форма: **Запросить концепт под ваш бренд**
- Действие одно: заполнить форму → получить концепт бесплатно

## Style direction
- **Tactile Editorial Premium**
- Редакционная сетка с крупной уверенной типографикой
- Чередование тёмных (#151716) и светлых (#f4efe8 / #ebe5dd) секций
- Коралловый (#ff6a3d) строго для CTA-элементов, не декоративно
- Серо-зелёный (#7c847d) для второстепенного текста, разделителей
- Тактильная product-фотография как главный визуальный якорь
- Без SaaS-стилистики, без градиентов, без glassmorphism
- Профессиональная credibility + физическая материальность объекта

## Brand inputs
- **Название:** DeStressToys
- **Знак в шапке:** DS
- **Позиционирование:** Кастомные бренд-объекты для компаний
- **Hero-слоган:** Бренд-объект, который не теряется среди обычного мерча
- **Логотип:** знак DS в header
- **Цвета (все используются):**
  - Тёмный: `#151716` — фоны акцентных секций, текст
  - Песочный: `#f4efe8` — основной светлый фон
  - Светлый: `#ebe5dd` — альтернативный светлый фон, карточки
  - Коралловый: `#ff6a3d` — только CTA-кнопки и акценты
  - Серо-зелёный: `#7c847d` — вторичный текст, иконки, разделители
- **Внешние референсы стиля:** отсутствуют

## Required sections (в порядке следования)
1. **Header** — лого DS, навигация, CTA-кнопка «Запросить концепт»
2. **Hero** — заголовок, подзаголовок, CTA, визуал продукта
3. **Comparison** — чем бренд-объект отличается от обычного мерча
4. **Why it works** — преимущества / почему это работает
5. **Texture** — тактильная деталь, материальность объекта
6. **Gallery** — примеры реализованных объектов
7. **Welcome Box** — применение: welcome kit / промонаборы
8. **Process** — как это работает (шаги: заявка → концепт → производство)
9. **Use cases** — кейсы применения (event, HR, партнёры, рассылки)
10. **Trust** — снижение риска, социальное доказательство
11. **FAQ** — частые вопросы
12. **Final CTA / Lead form** — форма «Запросить концепт под ваш бренд»
13. **Footer** — контакты, ссылки, copyright, ссылка на Privacy
14. **Privacy page** — отдельная страница (route /privacy)

## Content mode
- **Реальный копирайт**
- Копирайт пишется по ТЗ (см. ниже)
- Placeholder только там, где нужны реальные фото продукта (Gallery, Texture, Hero visual)

## Copywriting brief (ТЗ для копирайтера)

### Тон и голос
- Уверенный, профессиональный, без заигрывания
- B2B-язык: конкретика, польза, ROI от внимания
- Не «милый мерч», а «инструмент бренд-коммуникации»
- Без сленга, без восклицаний, без «уникальный/инновационный»

### Секция Hero
- H1: «Бренд-объект, который не теряется среди обычного мерча»
- Subhead: 1-2 предложения — что это такое и для кого
- Proof points: 3 коротких тезиса под кнопкой (иконка + текст)

### Секция Comparison
- Таблица или визуальный split: «Обычный мерч» vs «Бренд-объект DeStressToys»
- Параметры: время на столе, тактильный контакт, запоминаемость, кастомизация

### Секция Why it works
- 3–4 карточки преимуществ
- Каждая: заголовок + 1–2 предложения

### Секция Texture
- Описание материала и ощущения в руках
- Упор на тактильность как инструмент retention

### Секция Process
- 3–4 шага: Заявка → Концепт → Согласование → Производство
- Каждый шаг: название + 1 предложение

### Секция Use cases
- 4 кейса: event / welcome kit / партнёрский подарок / промо-рассылка
- Каждый: ситуация + что даёт бренд-объект

### Секция Trust
- Снятие возражений: минимальный тираж, сроки, качество, возврат
- Логотипы клиентов (placeholder: [LOGO 1–6])
- Цитата клиента (placeholder)

### Секция FAQ
- 5–7 вопросов: тираж, материалы, сроки, цена, форматы, логистика

### Final CTA форма
- Заголовок: «Запросить концепт под ваш бренд»
- Поля: Имя, Компания, Email, Описание задачи (textarea), кнопка «Отправить заявку»
- Microcopy под кнопкой: «Ответим в течение 1 рабочего дня. Концепт бесплатно.»

## Platform constraints
- React + Tailwind CSS (Vite)
- Routing: React Router (главная `/`, privacy `/privacy`)
- Без backend — форма отправляет на webhook или заглушка с success-state

## Responsive priority
- Desktop-first (основная аудитория работает с десктопа)
- Mobile должен быть корректным, не сломанным

## Hard constraints
- Нет тёмной темы как default (тёмные секции — это акцентные блоки, не тема)
- Нет glassmorphism
- Нет градиентов
- Нет иллюстраций (только product-фото и типографика)
- Нет анимаций кроме простых transition на hover
- Строго B2B-язык, без retail-tone

## Notes
- Форма на финальном CTA — ключевой элемент конверсии. Не убирать выше fold.
- Gallery и Texture секции — placeholder под реальные фото продукта. Codex ставит <img> заглушки.
- Privacy page должна быть минимальной, но корректной по структуре (GDPR-ready).
- Favicon: текст «DS» на тёмном фоне.
